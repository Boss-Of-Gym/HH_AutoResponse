# AutoResponseHH

Бот на **Playwright** для автоматических откликов на вакансии на [hh.ru](https://hh.ru).  
Управляется через веб-интерфейс — запустите `AutoResponseHH.exe` (или `python main.py`) и откроется окно с GUI.

---

## Быстрый старт на Windows

### Способ 1 — готовая сборка из репозитория (рекомендуется)

1. Скачайте папку `AutoResponseHH/` из репозитория (кнопка **Code → Download ZIP** или `git clone`)
2. Запустите `AutoResponseHH.exe`
3. При первом запуске программа автоматически скачает Chromium (~200 МБ) в папку `browsers/`  
   — дождитесь завершения скачивания, после этого откроется GUI
4. Заполните логин/пароль в разделе **Настройки → Авторизация** — готово

> `settings.json` и база данных `autoresponse.db` создаются рядом с `.exe`.  
> Все личные данные хранятся только локально и не попадают в git.

### Способ 2 — сборка из исходников

Потребуется Python 3.13+.

```bash
git clone https://github.com/Boss-Of-Gym/AutoResponseHH.git
cd AutoResponseHH

python -m venv .venv
.venv\Scripts\activate

pip install -r requirements.txt
playwright install chrome

# Запуск через Python напрямую
python main.py

# Или собрать exe
build.bat
# → dist\AutoResponseHH\AutoResponseHH.exe
```

## Возможности

### Поиск и фильтрация
- Поиск по нескольким запросам одновременно с указанием региона и диапазона опыта
- Предварительная фильтрация через HH.ru API (быстрее, без открытия каждой вакансии)
- Пропуск вакансий старше N дней
- Чёрный список компаний и лимит откликов на одну компанию за сессию
- Скоринг вакансий по зарплате и свежести

### Отклики
- Автовыбор резюме по ключевым словам в **заголовке** вакансии
- Персонализированные сопроводительные письма с подстановкой переменных из профиля
- Выбор шаблона письма по ключевым словам в **заголовке** вакансии
- Вакансии с дополнительными вопросами откладываются в очередь на ручной отклик
- Дедупликация — повторный отклик на ту же вакансию невозможен

### Лимиты и защита
- Учёт дневного лимита HH.ru (200 откликов/24ч): при его достижении время сохраняется в базу
- При следующем запуске — мгновенное уведомление с обратным отсчётом, браузер не открывается
- Лимит откликов за одну сессию — настраивается в GUI

### Сервисные функции
- Автоподнятие резюме в поиске (не чаще, чем раз в 4 часа)
- Проверка статусов откликов (просмотрено / приглашение / отказ)
- Планировщик для ежедневного запуска в заданное время
- Повтор при сетевых сбоях с задержкой между попытками
- CSV-лог откликов + скриншот при критической ошибке

### Интерфейс
- Веб-GUI на Flask — настройки, запуск, история откликов, статистика
- Стриминг логов в реальном времени прямо в браузере (читаемый текст, UTF-8)
- Редактор шаблонов писем с предпросмотром подстановок
- График активности за 7 дней, воронка откликов, топ компаний
- История с поиском и фильтрацией по статусу, экспорт в CSV

## Настройка

### settings.json

При первом запуске GUI создаёт `settings.json` рядом с программой автоматически.  
Заполните минимум в разделе **Настройки → Авторизация**:

```json
{
  "credentials": {
    "login": "ваш_телефон_или_email",
    "password": "ваш_пароль"
  }
}
```

Остальные параметры настраиваются через GUI.

### Выбор резюме и шаблона письма

Программа читает **только заголовок вакансии** — полный текст вакансии не открывается.  
Если в заголовке найдено ключевое слово из таблицы правил — используется соответствующее резюме/шаблон.  
Если ни одно правило не совпало — применяется значение по умолчанию.

Правила задаются в GUI в разделе **Резюме**:

| Ключевое слово в заголовке | Резюме |
|---|---|
| `automation`, `автоматизатор`, `auto qa`, `sdet` | Automation QA Engineer |
| `manual`, `ручной` | QA Engineer |
| *(ничего не совпало)* | резюме по умолчанию |

### Шаблоны сопроводительных писем

Создайте файлы в папке `cover_letters/` рядом с программой:

| Файл | Используется когда |
|---|---|
| `default.txt` | ни одно правило не совпало |
| `automation.txt` | заголовок содержит: automation, автоматизатор, sdet |
| `manual.txt` | заголовок содержит: manual, ручной |
| `qa_lead.txt` | заголовок содержит: senior, qa lead, ведущий |

В тексте доступны переменные:

| Переменная | Что подставляется |
|---|---|
| `{title}` | название вакансии |
| `{company}` | компания |
| `{company_clause}` | фраза " в компании X" (или пусто) |
| `{name}` | ваше имя |
| `{position}` | желаемая должность |
| `{years}` | лет опыта |
| `{key_skills}` | ключевые навыки |
| `{city}` | город |
| `{phone}` | телефон |
| `{email}` | email |
| `{github}` | ссылка на GitHub |
| `{portfolio}` | ссылка на портфолио |

## Запуск через CLI

```bash
# GUI (по умолчанию)
python main.py

# Полный запуск без GUI
python main.py --run

# Тестовый прогон (без реальных откликов)
python main.py --dry-run

# Проверить статусы откликов
python main.py --check-status

# Запускать ежедневно в заданное время
python main.py --run-at 08:30

# GUI на другом порту
python main.py --port 8080
```

## Что хранится локально (не попадает в git)

| Файл/папка | Содержимое |
|---|---|
| `settings.json` | логин, пароль, все настройки |
| `autoresponse.db` | история откликов, статусы, лог, состояние лимита |
| `cover_letters/*.txt` | шаблоны с личными данными |
| `response_log_*.csv` | хронология откликов по дням |

---

## База данных

SQLite (`autoresponse.db`), 5 таблиц:

| Таблица | Содержимое |
|---|---|
| `applied_vacancies` | URL, название, компания, дата отклика |
| `manual_review` | вакансии с доп. вопросами для ручного отклика |
| `response_log` | хронологический лог каждого отклика |
| `negotiations` | статусы откликов: ожидание / просмотрено / приглашение / отказ |
| `settings` | служебные значения (напр. время достижения лимита HH.ru) |

## Ограничения и ответственность

- HH.ru ограничивает отклики до **200 в сутки** на один аккаунт — бот отслеживает это, сохраняет время достижения лимита и при следующем запуске сразу сообщает, через сколько часов можно продолжить.
- Автоматизация действий на сайте может противоречить пользовательскому соглашению HH.ru — используйте на своё усмотрение и под собственную ответственность.
