# AutoResponseHH

Бот на **Playwright** для автоматических откликов на вакансии на [hh.ru](https://hh.ru).  
Управляется через веб-интерфейс — запустите `python main.py` и откроется браузер с GUI.

---

## Возможности

### Поиск и фильтрация
- Поиск по нескольким запросам одновременно с указанием региона и диапазона опыта
- Предварительная фильтрация через HH.ru API (быстрее, без открытия каждой вакансии)
- Пропуск вакансий старше N дней
- Чёрный список компаний и лимит откликов на одну компанию за сессию
- Скоринг вакансий по ключевым словам

### Отклики
- Автовыбор резюме по ключевым словам в названии вакансии
- Персонализированные сопроводительные письма с подстановкой переменных из профиля
- Вакансии с дополнительными вопросами откладываются в очередь на ручной отклик
- Учёт дневного лимита откликов HH.ru (200/24ч) и лимита сессии
- Дедупликация — повторный отклик на ту же вакансию невозможен

### Сервисные функции
- Автоподнятие резюме в поиске (не чаще, чем раз в 4 часа)
- Проверка статусов откликов (просмотрено / приглашение / отказ)
- Планировщик для ежедневного запуска в заданное время
- Повтор при сетевых сбоях с задержкой между попытками
- CSV-лог откликов + скриншот при критической ошибке

### Интерфейс
- Веб-GUI на Flask — настройки, запуск, история откликов, статистика
- Редактор шаблонов писем прямо в браузере с предпросмотром подстановок
- График активности за 7 дней, воронка откликов, топ компаний
- История с поиском и фильтрацией по статусу, экспорт в CSV
- Сборка в `.exe` через PyInstaller — `build.bat`

## Настройка

### 1. Создайте settings.json

```bash
copy settings.json.example settings.json   # Windows
# cp settings.json.example settings.json   # Linux/macOS
```

Откройте `settings.json` и заполните минимум:

```json
{
  "credentials": {
    "login": "ваш_телефон_или_email",
    "password": "ваш_пароль"
  }
}
```

Остальные параметры можно настроить в GUI.

### 2. Шаблоны сопроводительных писем

Создайте файлы в папке `cover_letters/`:

| Файл | Используется для |
|---|---|
| `default.txt` | все вакансии без специального правила |
| `automation.txt` | вакансии с ключевыми словами: automation, автоматизатор, sdet |
| `manual.txt` | ручное тестирование |
| `qa_lead.txt` | senior, qa lead, ведущий |

В тексте доступны переменные из профиля:

| Переменная | Что подставляется |
|---|---|
| `{title}` | название вакансии |
| `{company}` | компания |
| `{company_clause}` | фраза " в компании X" (или пусто) |
| `{name}` | ваше имя |
| `{position}` | желаемая должность |
| `{years}` | лет опыта |
| `{key_skills}` | ключевые навыки |
| `{city}` | город |
| `{phone}` | телефон |
| `{email}` | email |
| `{github}` | ссылка на GitHub |
| `{portfolio}` | ссылка на портфолио |

## Запуск

### GUI (рекомендуется)

```bash
python main.py
```

Откроется окно с веб-интерфейсом. Настройте профиль, поисковые запросы, шаблоны писем — и нажмите «Запустить». Логи выполнения стримятся в реальном времени.

По умолчанию сервер стартует на порту `5555`. Изменить порт:

```bash
python main.py --port 8080
```

### CLI

```bash
# Полный запуск (отправляет реальные отклики)
python main.py --run

# Тестовый режим (проходит вакансии, не откликается)
python main.py --dry-run

# Проверить статусы откликов (просмотрено / приглашение / отказ)
python main.py --check-status

# Запускать ежедневно в заданное время
python main.py --run-at 08:30
```

## Сборка в exe

```bash
build.bat
```

Результат: `dist\AutoResponseHH\AutoResponseHH.exe`

При первом запуске exe автоматически скачивает Chromium (~200 МБ) в папку `browsers/` рядом с exe. `settings.json` и база данных тоже создаются рядом с exe — данные пользователя хранятся локально и не требуют Python.

---

## Что хранится локально (не попадает в git)

| Файл/папка | Содержимое |
|---|---|
| `settings.json` | логин, пароль, все настройки |
| `autoresponse.db` | история откликов, статусы, лог |
| `cover_letters/*.txt` | шаблоны с личными данными |
| `response_log_*.csv` | хронология откликов по дням |

---

## База данных

SQLite (`autoresponse.db`), 4 таблицы:

| Таблица | Содержимое |
|---|---|
| `applied_vacancies` | URL, название, компания, дата отклика |
| `manual_review` | вакансии с доп. вопросами для ручного отклика |
| `response_log` | хронологический лог каждого отклика |
| `negotiations` | статусы откликов: ожидание / просмотрено / приглашение / отказ |

## Ограничения и ответственность

- HH.ru ограничивает отклики до **200 в сутки** на один аккаунт — бот отслеживает это и останавливается сам.
- Автоматизация действий на сайте может противоречить пользовательскому соглашению HH.ru — используйте на своё усмотрение и под собственную ответственность.
