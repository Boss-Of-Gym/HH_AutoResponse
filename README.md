# AutoResponseHH

Бот на **Playwright** для автоматических откликов на вакансии на [hh.ru](https://hh.ru).
Ищет вакансии по заданным запросам, фильтрует их по своим правилам и откликается вместо
рутинного ручного перебора десятков страниц поиска.

## Возможности

- 🔍 Поиск сразу по нескольким запросам, региону и диапазону опыта
- 📝 Персонализированное сопроводительное письмо с подстановкой `{title}` / `{company}`
- 📄 Автовыбор резюме под тип вакансии по ключевым словам в заголовке
- 🚫 Чёрный список компаний и лимит откликов на одну компанию за сессию
- 📅 Фильтр по свежести вакансии (пропуск устаревших объявлений)
- 🔁 Дедупликация — не откликается повторно на уже обработанные вакансии
- ⏳ Учёт дневного лимита откликов HH.ru (200/24ч) и лимита сессии
- 🖐 Вакансии с доп. вопросами работодателя откладываются в очередь на ручной отклик
- 🔌 Повтор при сетевых сбоях с задержкой между попытками
- 📊 CSV-лог и подробная статистика по каждому запросу в конце сессии
- 📸 Автоскриншот и трейс Playwright при падении

## Установка

```bash
git clone https://github.com/Boss-Of-Gym/HH_AutoResponse.git
cd HH_AutoResponse

python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # Linux/macOS

pip install -r requirements.txt
pip install playwright-stealth

```

## Настройка

### 1. Учётные данные

Создайте в корне проекта файл `.env`:

```env
login_number=your_phone_or_email
password=your_password
```

Файл в `.gitignore` и никогда не должен попадать в репозиторий.

### 2. Сопроводительное письмо

Создайте в корне `cover_letter.txt` (тоже в `.gitignore`, файл персональный). В тексте
можно использовать плейсхолдеры:

- `{title}` — название вакансии
- `{company}` — компания
- `{company_clause}` — готовая фраза вида " в компании X" (пусто, если компания неизвестна)

Если файла нет — бот просто откликается без сопроводительного письма.

### 3. Параметры поиска и лимиты

Всё остальное настраивается в [config/config.py](config/config.py):

| Параметр | Где | За что отвечает |
|---|---|---|
| `SearchConfig.QUERIES` | поиск | список поисковых запросов |
| `SearchConfig.AREA` | поиск | регион (`113` = Россия, `1` = Москва, `2` = СПб) |
| `SearchConfig.EXPERIENCE` | поиск | требуемый опыт |
| `BotConfig.MAX_RESPONSES_PER_RUN` | лимиты | лимит откликов за одну сессию |
| `BotConfig.MAX_PER_COMPANY` | лимиты | лимит откликов на одну компанию |
| `BotConfig.FRESHNESS_DAYS` | фильтры | пропускать вакансии старше N дней |
| `BotConfig.DELAY_MIN` / `DELAY_MAX` | поведение | случайная задержка между кликами |
| `BlacklistConfig.COMPANIES` | фильтры | компании, на которые не откликаться |
| `ResumeConfig.MATCH` | резюме | какое резюме выбирать по ключевым словам в вакансии |

## Запуск

```bash
python main.py
```

Открывается видимое (non-headless) окно Chrome — это осознанное решение, чтобы можно
было при необходимости вмешаться (капча, 2FA и т.п.). По завершении в консоль
выводится статистика сессии: сколько откликов отправлено, сколько пропущено и почему.

## Что бот создаёт при работе

| Файл/папка | Содержимое |
|---|---|
| `applied_vacancies.json` | история откликов — для дедупликации между запусками |
| `manual_review.json` | вакансии с доп. вопросами — требуют отклика вручную |
| `response_log_YYYY-MM-DD.csv` | лог всех откликов за день |
| `artifacts/screenshots/` | скриншот при критической ошибке |
| `artifacts/traces/` | трейс Playwright при критической ошибке |

Эти файлы — рабочие данные конкретного пользователя, поэтому они в `.gitignore` и не
хранятся в репозитории.

### Как посмотреть trace

```bash
playwright show-trace artifacts/traces/имя_файла.zip
```

Трейс-вьюер покажет шаги выполнения, DOM, скриншоты, навигацию и сетевые запросы —
удобно для разбора причины падения.

## Ограничения и ответственность

- HH.ru ограничивает отклики до **200 в сутки** — бот отслеживает это и останавливается
  сам, попытки обойти лимит не предусмотрены.
- Используйте на свой аккаунт и свою ответственность: автоматизация действий на сайте
  может противоречить пользовательскому соглашению HH.ru.
